# IoT Gateway and Smart Camera System

Edge computing and IoT gateway system using Raspberry Pi, Flask, sensors, and cloud-based AI services.

## Overview

This project was developed as part of the DT373B Systems Design for Internet of Things course at Kristianstad University. The system uses a Raspberry Pi as an IoT gateway and edge device to communicate with sensors and actuators, while providing remote monitoring and control through REST APIs and cloud services. Camera images and sensor data are processed locally and integrated with cloud-based AI services for event-driven operation.

## Features

* Configured a Raspberry Pi as an IoT gateway and edge device.
* Implemented a Flask-based REST API for communication and remote control.
* Integrated sensors, LEDs, and camera modules using GPIO interfaces.
* Enabled event-driven image capture and device control.
* Transferred sensor and image data between end devices, the gateway, and cloud services.
* Implemented automatic face recognition using cloud-based AI services.
* Triggered actions based on image analysis and sensor events.

## System Architecture

The system consists of:

* End devices connected through GPIO interfaces.
* Raspberry Pi gateway running Flask and REST APIs.
* Local edge processing on the gateway.
* Cloud-based AI services for image analysis.
* Event-driven control and monitoring.
